# Internationalization (i18n)

## Locale Routing

- Supported locales: `ja` (default), `en`
- Public routes: `/[locale]/path`
- Middleware detects locale from: cookie > Accept-Language > default (ja)
- Bare paths (e.g., `/about`) redirect to `/{locale}/about`

## Dictionary Structure

UI translations live in `src/dictionaries/{locale}.json`:

```json
{
  "common": { "home": "ホーム", ... },
  "auth": { "signIn": "サインイン", ... },
  "contact": { "title": "お問い合わせ", ... },
  "events": { "title": "イベント", ... },
  "metadata": { "title": "...", "description": "..." }
}
```

## Usage in Server Components

```tsx
import { getDictionary } from "@/lib/i18n/dictionaries";

export default async function Page({ params }) {
  const { locale } = await params;
  const dict = await getDictionary(locale);
  return <h1>{dict.common.siteName}</h1>;
}
```

## Bilingual Database Content

Database models use paired fields: `titleJa`/`titleEn`, `descriptionJa`/`descriptionEn`.
Japanese is always required; English is optional.
